PROJECT DESCRIPTION
================================================================================

MailMirror is a web application that allows users to securely share their Gmail 
inbox in a read-only mode via a public link.

Unlike Gmail's native sharing (which allows full access), this system ensures:
  • View-only access
  • No reply / edit / delete permissions
  • Controlled sharing via unique links
  • Safe, controlled, dynamic, and shareable experience

PROBLEM IT SOLVES
================================================================================

Gmail does not provide a way to share inbox as view-only.

Current options:
  ❌ Gmail delegation → full access (not safe)
  ❌ Screenshots / forwarding → not dynamic (manual)

MailMirror solution:
  ✔ Safe - Read-only Gmail scope, no write access
  ✔ Controlled - Unique links, ability to disable sharing
  ✔ Dynamic - Real-time email updates
  ✔ Shareable - One-click public link generation

HOW IT WORKS (USER FLOW)
================================================================================

1. USER LOGIN
   • User logs in using Google OAuth
   • Grants permission: gmail.readonly scope

2. TOKEN STORAGE
   • Backend stores securely:
     - Google ID
     - Email address
     - Refresh token (for auto-renewal)
   • Storage: PostgreSQL (Neon) + Prisma ORM

3. EMAIL FETCHING
   • Backend uses Gmail API with:
     - access_token (auto-refreshed via refresh_token)
     - read-only scope (no write access)

4. LINK GENERATION
   • User creates shareable link:
     - Example: https://yourapp.com/view/abc123
     - Unique token per user

5. PUBLIC VIEWING
   • Anyone with the link can:
     ✓ View email list
     ✓ Read email content
     ✗ Cannot reply
     ✗ Cannot delete
     ✗ Cannot modify

SYSTEM ARCHITECTURE
================================================================================

COMPONENT FLOW:

Frontend (React)
      ↓
Backend (Node.js + Express)
      ↓
Google OAuth (Authentication)
      ↓
Gmail API (Email fetching)
      ↓
Database (PostgreSQL + Prisma)
      ↓
Public Sharing Link

DATABASE SCHEMA (Prisma)
  • Users table: id, email, googleId, refreshToken, createdAt
  • ShareLinks table: id, userId, token, isActive, createdAt, expiresAt
  • Messages table: id, senderEmail, subject, body, timestamp (from Gmail)

API ENDPOINTS (Backend)
  POST /api/auth/google - OAuth callback
  GET /api/emails - Fetch user's emails (protected)
  POST /api/share - Generate share link (protected)
  DELETE /api/share/:id - Disable share link (protected)
  GET /api/view/:token - View shared inbox (public)
